# Deep System Audit — 2026-06-12

> Full adversarial audit by 7 parallel specialist agents: signal core, signal routes, ML pipeline, scheduler/paper engine, backend APIs, frontend, database models.

## Verdict
The signal pipeline has sound architectural bones — the four-pillar TA design, ML/TA conflict detection, and compression cap logic are all implemented correctly. However, two CRITICAL ML training flaws (calibration set reuse and tuner data leakage into the test split) mean the reported AUC figures are optimistic, and the ML weight calibration UI is making decisions based on inflated numbers. The single most important structural issue is that the paper trading engine can hold positions indefinitely when live prices are unavailable, creating phantom equity that silently inflates portfolio performance.

## Top 3 Priorities
1. Fix the ML calibrator/early-stopping data leakage (AUD-C2) and tuner test-set overlap (AUD-M9) — together they mean every model's reported AUC is optimistic, corrupting the weight calibration decisions that flow into every signal.
2. Fix `_refresh_market` swallowing all downstream jobs on ingestion failure (AUD-H5) — a transient yfinance blip currently silences all price alerts and paper trading for that slot with no fallback.
3. Fix the paper engine time-stop bypass when `live_price` is None (AUD-H10) — delisted or disrupted symbols are held forever at stale prices, inflating reported equity.

## What Is Working Well
- Four-pillar TA score uses `max()` within each pillar to avoid double-counting correlated indicators; death-cross hard override applied in correct order.
- ML/TA conflict detection uses a continuous graduated band with no cliff at 0.35 — correctly implemented.
- `_decide_style` threshold logic is mechanically correct; multi-style BUY differentiated downstream by compression filters.
- `_should_enter` correctly scores fusion result once after CB-5 fix; volume scored as genuinely independent signal.
- `_bulk_persist` read-before-write suppression of same-signal reinsertion correct in intent; paper engine signal reading uses 26-hour cutoff correctly.
- User-position auth filtering correct on every endpoint — `_fetch_pos` always enforces `user_id` scoping.
- Optuna background thread uses independent `SessionLocal` and snapshot-detaches ORM objects before session closes.

## CRITICAL Issues

### AUD-C1: Dead-Zone Label Bullish Bias Diluted by Recency Weight Blend *(Medium effort)*
**Subsystem:** ML Pipeline  
**Location:** `features/builder.py:build_features` line 313; `training/trainer.py` lines 160–162  
**Problem:** Binary label `fwd_ret > 0` in a bull-market window produces more `1` labels than `0`. `compute_sample_weight("balanced")` is applied but immediately blended with recency weights and renormalised, destroying the class-balance correction magnitude.  
**Fix:** Use symmetric label `np.where(fwd_ret >= threshold, 1, np.where(fwd_ret <= -threshold, 0, np.nan))` and drop NaN. Or apply class balancing after recency blend by rescaling per-class weights to sum equally.  
**Status:** ⬜ Open

### AUD-C2: Calibrator Fitted on Early-Stopping Validation Set — Optimistic Calibration *(Medium effort)*
**Subsystem:** ML Pipeline  
**Location:** `training/trainer.py:train_model` lines 275–291  
**Problem:** `X_cal` used simultaneously as XGBoost's `eval_set` for early stopping AND as the calibrator fitting set. XGBoost has already partially overfit to `X_cal`, so calibrated probabilities are optimistic and the precision-threshold search produces a too-permissive threshold OOS.  
**Fix:** Four-way split: train / early-stop / calibration / threshold-test. Reserve a separate 15% slice for early stopping, leaving `X_cal` as clean calibration set untouched by the fitting loop.  
**Status:** ⬜ Open

### AUD-C3: Paper Portfolio Has No User Scoping — All Users See All Portfolios *(Design decision)*
**Subsystem:** Backend API  
**Location:** `paper_portfolio.py:_get_portfolio`; `/list` endpoint  
**Problem:** `_get_portfolio` has no `user_id` filter. Any authenticated user can read any portfolio's full equity curve, trade history, and configuration.  
**Note:** Paper portfolios are a shared "house" account (admin-only writes). If this is intentional, add comment; if per-user design is wanted, requires full schema migration.  
**Fix:** Add explicit comment in `_get_portfolio` documenting intentional shared model.  
**Status:** ✅ Documented (intentional shared model)

## HIGH Issues

### AUD-H1: `adx_val=None` Coercion Bypasses ADX Guard — All Short-History Stocks Get Compression *(1-line)*
**Subsystem:** Signal Core  
**Location:** `signals.py:generate_all_signals` → `_make_signal()`  
**Problem:** `adx_val=float(reasons.get("adx") or 0.0)` → `None or 0.0 = 0.0` → always triggers choppy-market compression on short-history stocks.  
**Fix:** `adx_val=reasons.get("adx")` — the None-guard in `_apply_style_signal` line 1229 already handles None.  
**Status:** ✅ Fixed

### AUD-H2: Weekly Blend Inflates `fused_before_filters` Baseline — Asymmetric Compression Cap *(Small)*
**Subsystem:** Signal Core  
**Location:** `_apply_style_signal()` lines 1176–1185  
**Problem:** `fused_before_filters` snapshot taken AFTER weekly blend, making compression cap baseline ~3–7% higher for SWING/LONG styles.  
**Fix:** Take `fused_before_filters` immediately after ML+TA fusion, before weekly blend.  
**Status:** ✅ Fixed

### AUD-H3: `ml_test_auc=0.55` Stored When ML Service Down — Misleading *(1-line)*
**Subsystem:** Signal Core  
**Location:** `signals.py:_fetch_ml_data()` line 227  
**Problem:** Returns `(None, 0.55, {})` when ML service fails — AUC looks like a weak model while model is null.  
**Fix:** Return `0.0` as fallback AUC; add `ml_service_available` flag.  
**Status:** ✅ Fixed

### AUD-H4: Stored Signal Endpoint Silently Falls Back to SWING on Missing Style *(1-line)*
**Subsystem:** Signal Routes  
**Location:** `routes.py:signal_for` GET `/{symbol}`  
**Problem:** `stored.get(s_key) or stored.get("SWING")` returns wrong style silently when requested style has no stored row.  
**Fix:** Return 404 when style not found, or add `"fallback_style"` field.  
**Status:** ✅ Fixed

### AUD-H5: Single `try/except` in `_refresh_market` Kills Alerts and Paper Trading on Ingest Failure *(Small)*
**Subsystem:** Scheduler  
**Location:** `scheduler.py:_refresh_market`  
**Problem:** Transient yfinance timeout in `ingest_universe` prevents `check_signal_alerts()` and `paper_trading_step()` from running.  
**Fix:** Wrap each logical stage in its own `try/except`.  
**Status:** ✅ Fixed

### AUD-H6: `_bulk_persist` BUY→BUY Suppression Skips New Days — Stalls Stability Counter *(Small)*
**Subsystem:** Signal Routes  
**Location:** `routes.py:_bulk_persist`  
**Problem:** Same-signal check has no date guard, so consecutive BUY days are never inserted, stalling `_compute_stability` streak count.  
**Fix:** Add `last_row.ts.date() == date.today()` check before skipping.  
**Status:** ✅ Fixed

### AUD-H7: `_compute_stability` `max(..., 1)` Floor Misrepresents New Signals as Stable *(1-line)*
**Subsystem:** Signal Routes  
**Location:** `routes.py:_compute_stability`  
**Problem:** Always returns at least 1 even for brand-new signals with no history.  
**Fix:** Remove `max(..., 1)` floor; return `count` directly.  
**Status:** ✅ Fixed

### AUD-H8: OOS Suppression Uses Accuracy at 0.5 Threshold — Wrong Metric for Imbalanced Data *(1-line)*
**Subsystem:** ML Pipeline  
**Location:** `training/trainer.py:train_model` lines 235–236, 347  
**Problem:** CV accuracy gates OOS suppression, but accuracy is trivially high for majority-class prediction on imbalanced data.  
**Fix:** Gate on `cv_auc_mean < 0.52` instead of accuracy.  
**Status:** ✅ Fixed

### AUD-H9: Ensemble Three Uses Only XGBoost `buy_threshold` for All Models *(Small)*
**Subsystem:** ML Pipeline  
**Location:** `training/trainer.py:predict_latest_ensemble_three` line 587  
**Problem:** Ensemble converts blended probability to direction using XGBoost's threshold only, ignoring LGB and RF thresholds.  
**Fix:** Weight-average the three thresholds by the same AUC weights used for blending.  
**Status:** ✅ Fixed

### AUD-H10: Paper Engine Holds Positions Forever When `live_price` is Missing *(Small)*
**Subsystem:** Paper Engine  
**Location:** `paper_trading_engine.py:_monitor_positions` line 684  
**Problem:** `if not live_price: continue` skips ALL exit checks including time-stop.  
**Fix:** Separate price-dependent exits (stop/target/trail) from price-independent exits (time-stop, signal decay).  
**Status:** ✅ Fixed

### AUD-H11: Multi-Portfolio Regime Uses `portfolios[0]` VIX Thresholds for All *(Small)*
**Subsystem:** Paper Engine  
**Location:** `paper_trading_engine.py:paper_trading_step`  
**Problem:** Market regime classification driven by portfolio[0]'s config; thresholds written to all portfolios.  
**Fix:** Fetch regime using `_DEFAULT_CONFIG` only; let each portfolio apply overrides locally.  
**Status:** ✅ Fixed

### AUD-H12: `test_auc_mean` Is Average of Individual Model AUCs, Not True Ensemble OOS AUC *(Small)*
